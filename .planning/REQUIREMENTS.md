# Requirements: DocGuard Correctness & Consistency

**Defined:** 2026-06-02
**Core Value:** A developer running `docguard check` on any real FastAPI app gets accurate, deterministic results — zero false positives, zero silently missed endpoints.

## v1 Requirements

### Router Detection

- [ ] **ROUT-01**: DocGuard detects endpoints decorated by any router variable assigned to `APIRouter()` or `FastAPI()` in the codebase (not just variables named `app` or `router`)
- [ ] **ROUT-02**: User can explicitly list additional router variable names in `.docguard.yaml` via a `router_names` key as an escape hatch for edge cases
- [ ] **ROUT-03**: DocGuard resolves the full endpoint path for routes mounted via `app.include_router(sub_router, prefix="/prefix")` — paths in the drift report reflect the mounted path, not just the route-level path

### Spec Parsing

- [ ] **SPEC-01**: OpenAPI parameters defined via `$ref: '#/components/parameters/X'` are resolved and included in the drift comparison — they are not silently dropped as missing

### Type Accuracy

- [ ] **TYPE-01**: `Union[A, B]` fields (non-nullable) are represented as `"object"` in the comparison rather than silently collapsing to the type of the left branch, eliminating false TYPE_MISMATCH drift

### Reliability & Diagnostics

- [ ] **DIAG-01**: When a Python source file cannot be parsed (SyntaxError, OSError), DocGuard emits a warning to stderr identifying the file — it does not silently skip it
- [ ] **DIAG-02**: When DocGuard finds source files but detects zero endpoints, it emits a clear warning indicating a likely configuration issue (not exit-0 silence)
- [ ] **DIAG-03**: `severity_threshold` in `.docguard.yaml` filters which drift items affect the exit code — items below the threshold are shown but do not cause a non-zero exit

## v2 Requirements

### Spec Parsing (Advanced)

- **SPEC-02**: `allOf` schemas in OpenAPI specs (generated by Pydantic model inheritance) are merged before comparison — inherited fields are not treated as missing
- **SPEC-03**: External `$ref` (file references, remote URLs) emit a specific warning identifying which refs were skipped
- **SPEC-04**: `requestBodies` using `$ref` are resolved the same way as inline request bodies

### Type Accuracy (Advanced)

- **TYPE-02**: `Annotated[X, Query(...)]` fields extract the inner type `X` correctly — the `Query()` metadata does not corrupt the resolved type
- **TYPE-03**: `Literal["a", "b"]` fields are mapped to a JSON `"string"` enum type rather than a generic type

### Parser Improvements

- **PARSE-01**: `can_handle` falls back to checking Python source imports when no `requirements.txt` or `pyproject.toml` declares FastAPI
- **PARSE-02**: Endpoints decorated with `include_in_schema=False` are excluded from drift detection
- **PARSE-03**: Multi-level `include_router` chains (router mounted into router mounted into app) are resolved correctly

## Out of Scope

| Feature | Reason |
|---------|--------|
| LLM fixer improvements | Different risk profile, separate feature — LLM fixer is not in the correctness pipeline |
| New output formats | No user request; text/JSON/GitHub Actions covers existing needs |
| Non-FastAPI framework support | Scope creep; DocGuard is FastAPI-specific |
| Full external `$ref` resolution | Requires HTTP fetching and caching; v2 warning approach is sufficient for now |
| Interactive/watch mode | Not the tool's purpose |
| Performance optimization | Not a current pain point |
| `Depends` alias import tracking | Requires full import graph; too complex; document as known limitation |
